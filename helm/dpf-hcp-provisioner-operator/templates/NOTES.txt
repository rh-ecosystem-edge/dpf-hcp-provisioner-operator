NEXT STEPS:

1. Verify the operator is running:

   kubectl get pods -n {{ include "dpf-hcp-provisioner-operator.namespace" . }}

   You should see the controller-manager pod in a Running state.

2. Check operator logs:

   kubectl logs -n {{ include "dpf-hcp-provisioner-operator.namespace" . }} \
     -l control-plane=controller-manager -f

3. Verify the CRD is installed:

   kubectl get crd dpfhcpprovisioners.provisioning.dpu.hcp.io

4. Review operator configuration:

   The operator is configured via the DPFHCPProvisionerConfig CR:

   kubectl get dpfhcpconfig default -o yaml

   Fields not set in the CR use CRD defaults (see kubectl explain dpfhcpconfig.spec).

5. Create your first DPFHCPProvisioner custom resource:

   Example CR template:

   apiVersion: provisioning.dpu.hcp.io/v1alpha1
   kind: DPFHCPProvisioner
   metadata:
     name: my-dpu-cluster
     namespace: default
   spec:
     baseDomain: clusters.example.com
     dpuClusterRef:
       name: my-dpucluster
       namespace: default
     etcdStorageClass: ocs-storagecluster-ceph-rbd
     ocpReleaseImage: quay.io/openshift-release-dev/ocp-release:4.19.0-ec.5-x86_64
     pullSecretRef:
       name: my-pull-secret
     sshKeySecretRef:
       name: my-ssh-key
     virtualIP: 192.168.1.100
     controlPlaneAvailabilityPolicy: HighlyAvailable

   Apply it with:

   kubectl apply -f dpfhcpprovisioner.yaml

PREREQUISITES:

Ensure the following operators are installed on your management cluster:
- MCE Operator (Multi-Cluster Engine)
- DPF Operator (Data Processing Framework)
- HyperShift Operator
- ODF Operator (OpenShift Data Foundation)
- MetalLB Operator (if using LoadBalancer service type)

TROUBLESHOOTING:

- View operator events:
  kubectl get events -n {{ include "dpf-hcp-provisioner-operator.namespace" . }} --sort-by='.lastTimestamp'

- Check DPFHCPProvisioner status:
  kubectl get dpfhcpprovisioner -A
  kubectl describe dpfhcpprovisioner <name> -n <namespace>

- Common issues:
  * BlueField image not found: Ensure the image is in the registry with the correct OCP version tag
  * DPUCluster not found: Ensure DPUCluster CR exists and is referenced correctly
  * Pull secret missing: Create secret with '.dockerconfigjson' key in CR namespace
  * SSH key missing: Create secret with 'id_rsa.pub' key in CR namespace
